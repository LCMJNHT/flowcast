# FlowCast Demo UI 验收检查清单 v2.0

> 基于优化后的产品需求文档，此清单补充了异常流程、边界情况和体验细节的检查项。

使用方式：
- `P0`：内部 Demo 演示必须通过，失败即阻断验收
- `P1`：建议通过，失败不阻断首轮视图评审
- `P2`：体验优化项，可后续迭代

---

## 1. 全局体验

### 1.1 品牌与导航

- [ ] P0 App 名称显示为 `闪阅 FlowCast` 或在关键页面有明确品牌露出
- [ ] P0 底部导航包含 `视频 / 新闻 / 收银台 / 我的` 四个入口
- [ ] P0 App 默认进入或可立即到达 `视频` Tab
- [ ] P0 底部导航选中态清晰（文字加粗/颜色变化）
- [ ] P0 页面文案以中文为主，必要技术字段可保留英文
- [ ] P1 主色使用深墨绿/青绿（#0A7B67）
- [ ] P1 强调色使用暖金（#E3B341）用于金额和高价值信息
- [ ] P1 卡片圆角统一为 8dp
- [ ] P2 深色模式适配（后续版本）

### 1.2 交互一致性

- [ ] P0 所有核心卡片、按钮和返回入口可点击且不崩溃
- [ ] P0 二级页面返回后，用户能继续完成原模块或切换其他 Tab
- [ ] P0 断开 FastAPI 后，视频、新闻、支付、NFC 仍能展示本地 Mock 内容
- [ ] P0 页面加载时展示骨架屏或加载指示器
- [ ] P1 空态页面有友好的插图或文案
- [ ] P1 错误提示包含明确的重试入口
- [ ] P2 列表滚动时保持滚动位置（状态保持）

## 2. 视频 Feed 模块

### 2.1 主流程

- [ ] P0 可通过底部 `视频` Tab 进入
- [ ] P0 页面使用深色或沉浸式背景
- [ ] P0 内容区域接近竖屏视频卡比例（约 9:16）
- [ ] P0 每条视频至少展示标题、作者、来源标识
- [ ] P0 展示点赞、评论、分享等动作入口或明确占位
- [ ] P0 展示数据源状态（Mock / 公开视频源 / FastAPI）
- [ ] P0 视频不可播放时，有封面、渐变、播放图标或可识别的播放占位
- [ ] P0 页面不应退化成纯文本列表

### 2.2 异常流程

- [ ] P0 视频数据为空时，展示空态提示："暂无视频内容，稍后再来看看吧"
- [ ] P0 视频加载失败时，展示重试按钮："加载失败，点击重试"
- [ ] P0 后端不可用时，自动降级到本地 Mock（无感知切换）

### 2.3 加载状态

- [ ] P0 首屏加载时展示骨架屏（深色背景 + 渐变占位）
- [ ] P0 下拉刷新时展示标准 Compose 刷新指示器
- [ ] P1 滚动到底部自动加载，展示 loading 指示器
- [ ] P1 加载失败时展示错误提示和重试按钮

### 2.4 视觉细节

- [ ] P0 顶部频道栏展示：推荐/AI/零售/NFC/支付（横向滚动）
- [ ] P0 频道标签选中态清晰（白色背景 + 深色文字）
- [ ] P1 视频卡背景使用渐变（#255E5D → #111A17 → #090D0C）
- [ ] P1 互动按钮（赞/评/转）使用半透明白色背景
- [ ] P2 视频卡支持上下滑动切换（后续版本）

## 3. 新闻模块

### 3.1 新闻列表

- [ ] P0 可通过底部 `新闻` Tab 进入
- [ ] P0 顶部明确当前频道为 AI 新闻或科技资讯
- [ ] P0 每张新闻卡片展示标题、概要、来源、发布时间
- [ ] P0 标题比概要更突出（标题 17sp 加粗，概要 14sp 灰色）
- [ ] P0 概要不堆长段落（最多 3 行，超出省略）
- [ ] P0 卡片点击后进入新闻详情页
- [ ] P0 列表为空或接口失败时有 Mock 兜底或空态提示
- [ ] P0 页面整体为清爽浅色阅读风格，不使用视频页的沉浸黑背景

### 3.2 频道筛选

- [ ] P0 顶部频道筛选栏可横向滚动
- [ ] P0 至少包含：全部/AI 产品/开发者/零售科技
- [ ] P0 选中态与未选中态可区分
- [ ] P1 频道切换有平滑过渡动画
- [ ] P1 空频道展示占位提示："该频道暂无内容"

### 3.3 新闻详情

- [ ] P0 顶部展示新闻标题和来源/发布时间信息
- [ ] P0 主体展示 AI 总结，不展示未授权全文转载
- [ ] P0 至少展示 3 条关键点，便于评审者快速理解内容
- [ ] P0 页面展示原文链接、链接信息块或打开原文按钮
- [ ] P0 详情页提供返回入口
- [ ] P1 原文链接点击后有明确跳转行为或模拟跳转反馈
- [ ] P1 返回新闻列表后，底部导航状态仍保持在新闻模块

### 3.4 异常流程

- [ ] P0 新闻不存在时展示缺省页："新闻不存在，请返回列表重新选择"
- [ ] P0 原文链接失效时展示提示："原文链接已失效"
- [ ] P0 内容加载失败时展示重试："内容加载失败，点击重试"

## 4. 收银台模块

### 4.1 收银台页

- [ ] P0 可通过底部 `收银台` Tab 进入
- [ ] P0 可由 NFC 收款 Scheme 解析页进入或跳回收银台
- [ ] P0 页面突出展示订单金额（大字号，金色）
- [ ] P0 展示商户名称、订单号或业务场景
- [ ] P0 至少提供模拟支付宝、模拟微信、模拟银联三种方式
- [ ] P0 页面展示当前收银台 Scheme 或订单来源
- [ ] P0 支付方式有选中态（边框高亮 + 背景色）
- [ ] P0 主按钮文案明确，例如 `确认支付` 或 `使用模拟微信支付`
- [ ] P0 点击主支付按钮进入对应 MockPaymentScreen

### 4.2 异常流程

- [ ] P0 订单创建失败时展示错误提示："订单创建失败，请重试"
- [ ] P0 支付方式不可用时置灰或隐藏
- [ ] P0 金额为 0 或负数时拦截提交："订单金额无效"

### 4.3 Mock 支付页

- [ ] P0 页面标题或品牌区明确当前模拟渠道：支付宝、微信或银联
- [ ] P0 展示订单金额
- [ ] P0 展示支付 Scheme、订单号或其他可解释当前交易的信息
- [ ] P0 提供 `支付成功`、`支付失败`、`取消支付` 三个结果动作
- [ ] P0 三个动作都能跳转到 PaymentResultScreen
- [ ] P0 返回或取消时不应造成导航栈卡死
- [ ] P1 页面视觉应与收银台区分，但不冒充真实支付 App
- [ ] P1 可在文案中说明"仅模拟支付结果，不接真实渠道"

## 5. 支付结果模块

### 5.1 结果展示

- [ ] P0 支持成功、失败、取消状态展示
- [ ] P0 成功状态使用积极颜色（绿色 #158A55）和明确文案
- [ ] P0 失败/取消状态与成功状态可区分（失败红色，取消橙色）
- [ ] P0 提供返回收银台或继续操作入口
- [ ] P0 从结果页返回后，用户能继续浏览其他 Tab

### 5.2 状态详情

| 状态 | 图标 | 颜色 | 文案 |
|------|------|------|------|
| success | ✓ | 绿色 | "支付成功" |
| failed | ! | 红色 | "支付失败" |
| canceled | ! | 橙色 | "支付已取消" |
| processing | … | 蓝色 | "支付处理中" |

- [ ] P0 展示订单号、金额、支付方式中的至少两个关键信息
- [ ] P1 支持处理中状态的展示口径

## 6. NFC 模块

### 6.1 NFC Demo 页

- [ ] P0 可从 `我的` 页面进入
- [ ] P0 页面包含三类场景：线下商户收款、门店桌牌点餐、小程序跳转
- [ ] P0 每个场景展示简短业务说明和对应 Scheme
- [ ] P0 每个场景可点击触发解析或跳转
- [ ] P0 页面应突出"碰一碰触发"或 NDEF URI 读取后的 Demo 语境
- [ ] P0 不应只展示裸字符串列表
- [ ] P1 每个场景有可区分标签，例如收款/点餐/小程序

### 6.2 NFC 解析页

- [ ] P0 展示原始 Scheme
- [ ] P0 展示业务动作说明：进入收银台/进入点餐页/进入小程序跳转页
- [ ] P0 展示解析后的关键字段（merchantId、amount、storeId、tableId 等）
- [ ] P0 线下收款场景可从解析页进入收银台
- [ ] P0 桌牌点餐和小程序跳转场景至少展示下一步占位说明
- [ ] P0 从解析页可以返回 NFC Demo 或继续完成后续动作
- [ ] P1 无法识别的 Scheme 应展示错误提示，不应崩溃

### 6.3 桌牌点餐页（占位）

- [ ] P0 展示门店名称（如 FlowCast Coffee）
- [ ] P0 展示桌号（如 T08 桌）
- [ ] P0 展示已从 NFC Scheme 恢复 storeId 与 tableId 的说明
- [ ] P1 展示推荐菜单（2-3 个示例菜品）
- [ ] P1 点餐入口占位按钮

### 6.4 小程序跳转页（占位）

- [ ] P0 展示平台（wechat/alipay/douyin）
- [ ] P0 展示 AppId
- [ ] P0 展示页面路径
- [ ] P0 说明"Demo 不真实拉起外部平台，仅展示参数解析结果"

### 6.5 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| Scheme 无法识别 | 展示错误状态 | "当前 Scheme 暂不支持" |
| 缺少必填字段 | 展示错误详情 | "Scheme 缺少必要参数" |
| 解析失败 | 展示原始 Scheme | "解析失败，请检查 Scheme 格式" |

- [ ] P0 以上异常场景均有对应处理，不崩溃

- [ ] P0 展示能力概览：短视频、AI 新闻、模拟支付、NFC

## 8. 后端契约冒烟测试

若 FastAPI Mock 启动，应满足以下契约：

- [ ] P0 `GET /health` 返回 `status=ok`，并可附带 service、version、mode、timestamp 等 Mock 元信息
- [ ] P0 `GET /videos/feed` 返回 `items` 和 `next_cursor`，每项包含 title、author、source、cover_url/play_url 或可解释占位
- [ ] P0 `GET /news` 返回新闻列表；`GET /news/{id}` 返回 summary、key_points、source_url
- [ ] P0 `POST /orders` 返回 pending 订单、amount、merchant 信息和 cashier_scheme
- [ ] P0 `GET /payments/methods` 返回 alipay、wechat、unionpay
- [ ] P0 `POST /payments/mock` 能把订单更新为 success、failed 或 canceled
- [ ] P0 `GET /payments/status/{order_id}` 返回 order_id、status、method、amount
- [ ] P0 `GET /nfc/scenarios` 返回 payment、order、miniapp 三类场景和 Scheme

## 9. Demo 演示验收

评审演示必须能按以下路径完成：

```
视频 Tab
  -> 新闻 Tab
  -> 新闻详情
  -> 返回新闻
  -> 收银台
  -> 选择一种模拟支付方式
  -> 模拟支付成功/失败/取消任一状态
  -> 支付结果
  -> 我的
  -> NFC Demo
  -> 线下收款 Scheme
  -> Scheme 解析
  -> 收银台
```

通过标准：

- [ ] P0 全流程无崩溃
- [ ] P0 每一步页面目标清晰
- [ ] P0 所有 Mock 状态都能被解释为第一版 Demo 行为
- [ ] P0 评审者不看代码也能说出四条主路径：视频、新闻、支付、NFC
- [ ] P1 再走一遍 NFC 桌牌点餐和小程序场景，确认占位动作说明清楚

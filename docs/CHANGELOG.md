## 变更记录格式说明

### 版本号规则

采用语义化版本号：`主版本号。次版本号.修订号`

- 主版本号：不兼容的 API 修改或产品定位变更
- 次版本号：向下兼容的功能性新增
- 修订号：向下兼容的问题修正

### 变更类型

- **新增**：新功能、新模块、新接口
- **变更**：现有功能的修改
- **优化**：性能、体验、代码质量改进
- **删除**：移除的功能或接口
- **修复**：Bug 修复

## 决策记录

### 决策 #1：Mock 优先策略

**时间**: 2026-05-14  
**决策内容**: 第一版优先使用 Mock 数据，确保视图和交互闭环  
**原因**: 快速验证产品形态，降低前期开发成本  
**影响范围**: Android 本地 Mock + FastAPI Mock 双轨并行

### 决策 #2：Scheme 设计规范

**时间**: 2026-05-14  
**决策内容**: 统一使用 `flowcast://` 作为 Scheme 协议头  
**原因**: 避免与真实支付渠道 Scheme 冲突，便于 Demo 环境识别  
**影响范围**: 所有支付和 NFC 场景

### 决策 #3：降级策略

**时间**: 2026-05-14  
**决策内容**: 后端不可用时，客户端自动降级到本地 Mock  
**原因**: 确保 Demo 在任何情况下都可演示  
**影响范围**: Android 数据层

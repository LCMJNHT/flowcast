# FlowCast Demo 产品需求文档 v2.0

> 本文档在原有 PRD 基础上，补充了异常流程、边界情况、加载状态和空态处理等细节定义。

## 1. 产品定位与目标

### 1.1 产品定位

FlowCast Demo 是一个内部原型，用于验证"短视频 + AI 新闻 + Scheme 模拟支付 + NFC 场景跳转"的一体化产品形态。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 视图优先 | 先确认页面结构、视觉密度和交互闭环 |
| Mock 优先 | 所有数据源允许 Mock，但接口契约保持生产形态 |
| 可演示性 | 每个核心入口必须可点击、可返回、可解释 |
| 可降级 | 后端不可用时，Android 仍能展示核心页面 |

### 1.3 目标用户

- 内部产品、研发、设计评审人员
- 线下演示场景中的业务方观察者
- 执行手工 UI 验收和冒烟测试的 QA/Agent

## 2. 功能需求详解

### 2.1 视频 Feed 模块

#### 2.1.1 主流程

```
打开 App → 默认进入视频 Tab → 浏览沉浸式视频 Feed → 查看作者/标题/来源/互动
```

#### 2.1.2 页面元素

| 元素 | 必填 | 说明 | 占位规则 |
|------|------|------|----------|
| 视频封面/背景 | P0 | 沉浸式竖屏卡片 | 渐变色背景 + 播放图标 |
| 标题 | P0 | 视频标题 | 最多 2 行，超出省略 |
| 作者 | P0 | 创作者名称 | 带@前缀 |
| 来源标识 | P0 | 数据源状态说明 | Mock/Public Video API/FastAPI |
| 描述 | P0 | 视频简介 | 最多 2 行 |
| 点赞数 | P0 | 互动数据 | 格式如 12.8k |
| 评论数 | P0 | 互动数据 | 格式如 326 |
| 分享入口 | P1 | 占位按钮 | 展示图标，点击 toast 提示 |

#### 2.1.3 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| 视频数据为空 | 展示空态提示 | "暂无视频内容，稍后再来看看吧" |
| 视频加载失败 | 展示重试按钮 | "加载失败，点击重试" |
| 后端不可用 | 自动降级到本地 Mock | 无感知切换 |

#### 2.1.4 加载状态

| 状态 | 展示 |
|------|------|
| 首屏加载 | 骨架屏（深色背景 + 渐变占位） |
| 下拉刷新 | 标准 Compose 刷新指示器 |
| 加载更多 | 滚动到底部自动加载，展示loading 指示器 |

### 2.2 新闻模块

#### 2.2.1 新闻列表

##### 主流程

```
点击新闻 Tab → 浏览 AI 新闻卡片 → 点击卡片 → 进入详情页
```

##### 页面元素

| 元素 | 必填 | 说明 |
|------|------|------|
| 频道标签 | P0 | 全部/AI 产品/开发者/零售科技（横向滚动） |
| 新闻卡片 | P0 | 标题 + 摘要 + 来源 + 时间 |
| 空态 | P0 | 无新闻时展示占位提示 |

##### 卡片信息层级

```
┌─────────────────────────────┐
│ [来源标签]  [发布时间]       │  ← 第一行
├─────────────────────────────┤
│ 新闻标题（加粗，17sp）       │  ← 第二行
│ 新闻摘要（灰色，14sp，3 行）  │  ← 第三行
│ [查看 AI 总结]               │  ← 行动点
└─────────────────────────────┘
```

##### 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| 列表为空 | 展示空态卡片 | "暂无新闻内容" |
| 加载失败 | 展示重试按钮 | "加载失败，点击重试" |
| 频道切换 | 无数据时展示占位 | "该频道暂无内容" |

#### 2.2.2 新闻详情

##### 页面元素

| 元素 | 必填 | 说明 |
|------|------|------|
| 标题 | P0 | 与列表页一致 |
| 来源信息 | P0 | 来源 + 发布时间 |
| AI 总结 | P0 | 优先展示，大字号 |
| 关键点 | P0 | 编号列表，至少 3 条 |
| 原文链接 | P0 | 可点击跳转或模拟跳转 |

##### 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| 新闻不存在 | 展示缺省页 | "新闻不存在，请返回列表重新选择" |
| 原文链接失效 | 展示提示 | "原文链接已失效" |
| 加载失败 | 展示重试 | "内容加载失败，点击重试" |

### 2.3 收银台模块

#### 2.3.1 主流程

```
进入收银台 → 查看商户/订单/金额 → 选择支付方式 → 确认支付 → Mock 支付页 → 支付结果
```

#### 2.3.2 页面元素

##### 收银台页

| 元素 | 必填 | 说明 |
|------|------|------|
| 商户名称 | P0 | 如"FlowCast 线下门店" |
| 订单号 | P0 | 如 O10001 |
| 金额 | P0 | 大字号，金色强调 |
| 支付方式列表 | P0 | 支付宝/微信/银联，单选 |
| 收银台 Scheme | P0 | 技术字段展示 |
| 确认支付按钮 | P0 | 使用选中支付方式文案 |

##### 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| 订单创建失败 | 展示错误提示 | "订单创建失败，请重试" |
| 支付方式不可用 | 置灰或隐藏 | 不展示不可用方式 |
| 金额为 0 或负数 | 拦截提交 | "订单金额无效" |

#### 2.3.3 Mock 支付页

| 元素 | 必填 | 说明 |
|------|------|------|
| 渠道名称 | P0 | 如"模拟支付宝" |
| 金额 | P0 | 与收银台一致 |
| 支付 Scheme | P0 | 技术字段展示 |
| 支付成功按钮 | P0 | 主按钮 |
| 支付失败按钮 | P0 | 次要按钮 |
| 取消支付按钮 | P0 | 次要按钮 |

#### 2.3.4 支付结果页

| 状态 | 图标 | 颜色 | 文案 |
|------|------|------|------|
| success | ✓ | 绿色 | "支付成功" |
| failed | ! | 红色 | "支付失败" |
| canceled | ! | 橙色 | "支付已取消" |
| processing | … | 蓝色 | "支付处理中" |

##### 结果页元素

- 订单号
- 金额
- 支付方式
- 返回收银台按钮

##### 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| 状态未知 | 展示处理中 | "订单状态等待确认中" |
| 订单不存在 | 展示错误 | "订单不存在，请返回重试" |

### 2.4 NFC 模块

#### 2.4.1 NFC Demo 页

##### 页面元素

| 元素 | 必填 | 说明 |
|------|------|------|
| 场景说明 | P0 | 碰一碰触发，模拟 NDEF URI 读取 |
| 场景列表 | P0 | 线下商户收款/桌牌点餐/小程序跳转 |
| 场景卡片 | P0 | 标题 + 描述 + Scheme |

#### 2.4.2 NFC 解析页

##### 页面元素

| 元素 | 必填 | 说明 |
|------|------|------|
| 原始 Scheme | P0 | 完整展示 |
| 解析字段 | P0 | key-value 列表 |
| 业务动作 | P0 | 说明下一步行为 |
| 跳转按钮 | P0 | 根据场景类型变化 |

##### 场景类型与动作

| 场景 ID | 类型 | 动作 | 目标页 |
|---------|------|------|--------|
| merchant | payment | open_cashier | CashierScreen |
| table | order | open_table_order | TableOrderScreen |
| miniapp | miniapp | launch_miniapp | MiniProgramJumpScreen |

##### 异常流程

| 场景 | 表现 | 文案 |
|------|------|------|
| Scheme 无法识别 | 展示错误状态 | "当前 Scheme 暂不支持" |
| 缺少必填字段 | 展示错误详情 | "Scheme 缺少必要参数" |
| 解析失败 | 展示原始 Scheme | "解析失败，请检查 Scheme 格式" |

| 推荐菜单 | P1 | 2-3 个示例菜品 |
| 点餐入口 | P1 | 占位按钮 |

#### 2.4.4 小程序跳转页（占位）

| 元素 | 必填 | 说明 |
|------|------|------|
| 平台 | P0 | wechat/alipay/douyin |
| AppId | P0 | 小程序 ID |
| 路径 | P0 | 页面路径 |
| 跳转说明 | P0 | Demo 不真实拉起外部平台 |

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载时间 | < 2s |
| 页面切换流畅度 | 无明显卡顿 |
| 列表滚动帧率 | 稳定 60fps |

### 3.2 兼容性要求

| 项目 | 要求 |
|------|------|
| 最低 Android 版本 | API 24 (Android 7.0) |
| 推荐 Android 版本 | API 29+ (Android 10+) |
| 屏幕适配 | 手机竖屏，1080x2400 基准 |

### 3.3 可访问性要求

| 项目 | 要求 |
|------|------|
| 字体大小 | 支持系统字体缩放 |
| 颜色对比度 | WCAG AA 标准 |
| 内容描述 | 关键元素有 contentDescription |

## 5. 状态管理

### 5.1 支付状态机

```
pending ──→ success
   ├──→ failed
   └──→ canceled
```

### 5.2 订单状态机

```
created ──→ pending ──→ success
              ├──→ failed
              └──→ canceled
```

### 5.3 NFC 场景状态

| 状态 | 说明 | 处理 |
|------|------|------|
| valid | 有效 Scheme | 正常解析并跳转 |
| invalid | 无效 Scheme | 展示错误状态 |
| unsupported | 不支持的类型 | 展示不支持提示 |

## 6. 错误处理

### 6.1 网络错误

| 场景 | 处理 |
|------|------|
| 后端不可达 | 自动降级到本地 Mock |
| 请求超时 | 展示超时提示，提供重试 |
| 响应格式错误 | 记录日志，展示通用错误 |

### 6.2 路由错误

| 场景 | 处理 |
|------|------|
| 目标页不存在 | 展示缺省页，提供返回按钮 |
| 参数缺失 | 展示错误信息，提供返回按钮 |
| 导航栈异常 | 重置到首页 |

### 6.3 数据错误

| 场景 | 处理 |
|------|------|
| 空数据 | 展示空态占位 |
| 数据解析失败 | 使用默认值或展示错误 |
| 必填字段缺失 | 展示错误提示 |

- [ ] 组件样式统一

## 8. 演示脚本

```
1. 打开 App → 视频 Feed（深色沉浸）
2. 切换到新闻 Tab → 进入新闻详情 → 查看 AI 总结 → 返回
3. 切换到收银台 → 选择模拟微信支付 → 支付成功 → 返回
4. 切换到我的 → NFC Demo → 线下商户收款 → Scheme 解析 → 进入收银台
5. NFC Demo → 桌牌点餐/小程序跳转 → 查看占位说明
```
